    這個lwext4項目的主要目標是為微控制器提供 ext2/3/4 文件系統。 它可能是傳統的MCU 文件系統 庫( 大多數基於 FAT32 )的一個有趣的替代方案。 庫在微控制器世界中具有一些很酷和獨特的特性：

    • 目錄索引- 快速文件查找和列表操作
    • 範圍- 快速大文件截斷
    • 日誌事務處理&恢復- 功耗阻力

    Lwext4是 sd/mmc卡，USB Flash 驅動器或者任何其他 Wear 內存類型的最佳選擇。 然而，它對原始 Flash 設備並不適用。

    • xattr支持
    • 元數據校驗和支持
    • 日誌恢復&事務
    • 許多錯誤修正&改進

    Lwext4也可以用作fuse內部組件。 下面是一個使用lwext4作為 文件系統 基礎的不錯的項目：

    • https://github.com/ngkaho1234/fuse-lwext4

    某些源文件是在GPLv2下授權的。 它使整個 lwext4 GPLv2授權。 要將庫用作 BSD3，必須先刪除GPLv2許可的源文件。 此時，有兩個在GPLv2下許可的文件：

    • ext4_xattr.c
    • ext4_extents.c

    所有其他模塊和頭文件都是BSD-3-Clause許可代碼。

    特性
    • 文件類型：常規目錄，目錄，softlinks
    • 對hardlinks的支持
    • 支持多個 blocksize: 1 KB，2 KB，4KB。 64 KB
    • 支持小/大的前端架構
    • 多重配置 (ext2/ext3/ext4)
    • 僅C 標準庫依賴項
    • 支持多種CPU體系結構( x86/64，cortex，。)
    • 內存佔用小
    • 靈活的配置

    內存佔用

    高級的ext4 文件系統 特性，比如擴展或者日誌，需要一些內存。 但是，大部分昂貴的內存功能在編譯時都可以被禁用。 以下是cortex-m4處理器的簡要摘要：

    • 文本：20KB - 只有 ext2 fs支持，50KB - 全fs特性集
    • 數據：8KB 最小 8 x 1KB 塊緩存，當啟用日誌記錄和擴展時 12KB
    • 堆棧：2KB - 足夠( 不精確測量)

    在不需要malloc的情況下，可以動態分配模塊的先前版本，但是需要從 1.0.0動態內存分配。 但是，塊緩存不應分配超過 CONFIG_BLOCK_DEV CACHE_SIZE。

    支持的ext2/3/4 特性

    不兼容：

    • 文件，恢復，meta_bg，擴展區，64位，flex_bg:
    • 壓縮，journal_dev，mmp，ea_inode，DirData，bg_meta_csum，largedir，inline_data:

    兼容：

    • has_journal，ext_attr，dir_index:
    • dir_prealloc，imagic_inodes，resize_inode:

    只讀：

    • sparse_super，large_file，huge_file，gdt_csum，dir_nlink，extra_isize，metadata_csum:
    • 配額，bigalloc，btree_dir:

    編譯

    依賴項

    • Windows

    下載 MSYS-2: https://sourceforge.net/projects/msys2/

    安裝所需軟件包是 MSYS2 shell 軟件包管理器：

    



 pacman -S make gcc cmake p7zip
    • Linux

    軟件包安裝( Debian ):

    



 apt-get install make gcc cmake p7zip

    編譯&安裝工具

    



 make generic


 cd build_generic


 make


 sudo make install
    lwext4-generic演示應用程序

    簡單lwext4庫測試應用程序：

    • 加載 ext2/3/4 圖像
    • 用 ext2/3/4 部件加載linux塊設備
    • 使用 ext2/3/4 文件系統 加載 Windows 卷
    • 目錄速度測試
    • 文件寫入/讀取速度測試

    如何使用圖像/blockdevices:

    



 lwext4-generic -i ext_images/ext2 


 lwext4-generic -i ext_images/ext3 


 lwext4-generic -i ext_images/ext4

    顯示完整選項集：

    



 lwext4-generic --help
    運行自動測試

    對 100MB 個已經解壓映像執行測試：

    



 make test

    執行自動生成的1GB 映像( 僅在Linux目標上) + fsck的測試：

    



 make test_all
    使用lwext4-mkfs工具

    使用內部庫工具創建 ext2/3/4 分區是有可能的。

    生成空文件( 1gb ):

    



 dd if=/dev/zero of=ext_image bs=1M count=1024

    創建ext2分區：

    



 lwext4-mkfs -i ext_image -e 2

    創建ext3分區：

    



 lwext4-mkfs -i ext_image -e 3

    創建ext4分區：

    



 lwext4-mkfs -i ext_image -e 4

    顯示完整選項集：

    



 lwext4-mkfs --help
    交叉編譯獨立庫

    需要 Toolchains:

    可以為許多目標編譯 Lwext4. 以下是 8位/16/32/64 位架構的示例。

    • x86或者amd64通用
    • 用於 ARM cortex-m0/m3/m4 微控制器的arm-none-eabi-gcc
    • AVR單片機用於 AVR xmega微控制器
    • 用於blackfin處理器的bfin-elf-gcc
    • 適用於msp430微控制器的msp430-gcc

    庫只對通用( amd64 ) & ARM Cortex M 體系結構進行了測試。 其他目標編譯通過( 帶警告的地方) 但測試還未完成。 Lwext4代碼是用endianes編寫的。 還未測試大的前端行為。

    構建avrxmega7庫：

    



 make avrxmega7


 cd build_avrxmega7


 make lwext4

    構建cortex-m0庫：

    



 make cortex-m0


 cd build_cortex-m0


 make lwext4

    構建cortex-m3庫：

    



 make cortex-m3


 cd build_cortex-m3


 make lwext4

    構建cortex-m4庫：

    



 make cortex-m4


 cd build_cortex-m4


 make lwext4
